## Summary (≤ 200 words)

Implemented the three code deliverables and their tests for the fail-closed escalation policy.

`EscalationVerdict` is a plain positional record — no logic, just the wire type.

`FailClosedEscalator` is a static class with two `Evaluate` overloads (one for `SonnetResult`, one for `HaikuResult`), a `MostSevere` helper for mission-level aggregation (AT-03), and the private `ApplyStateMachine` that implements the exact switch from WP-11's state machine spec. The optional `worktreeDiff` parameter on the `SonnetResult` overload wires in the banned-pattern check so AT-05 is testable without external glue code.

`BannedPatternDetector` parses standard git unified-diff output line by line, tracking the current file from `+++ b/` headers, and runs six pattern checks on every `+` (addition) line. Patterns 1–2 are project-scoped (allowed in `Warden.Anthropic` / `Warden.Orchestrator`); Pattern 3 uses an `ECSCli` presence test on the same line; Pattern 4 exempts `Warden.Orchestrator/Program.cs`; Pattern 5 matches `<PackageReference>`/`<ProjectReference>` in three named csproj files; Pattern 6 pairs `Task.Run` with a set of orchestrator type-name fragments.
