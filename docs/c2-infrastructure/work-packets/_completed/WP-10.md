## Summary (≤ 200 words)

Implemented the concrete `IChainOfThoughtStore` and the full `./runs/<runId>/` persistence tree described in SRD §2 Pillar B.

`RunId.Generate()` produces `{yyyyMMdd-HHmmss}-{slug}` identifiers; the existing `RunCommand.GenerateRunId()` helper was replaced with a delegation to it. `RunLayout` is a single-source-of-truth class for every path under the run root — consumers build paths through it so renames cost one edit.

`ChainOfThoughtStore` implements `IChainOfThoughtStore` (four Sonnet artefact methods) plus two extension methods: `PersistMissionAsync` for `mission.md` and `AppendEventAsync` for the append-only `events.jsonl`. The events gate uses a `SemaphoreSlim` so concurrent Sonnet workers cannot interleave partial lines. `InitRun` checks for directory existence before creation, providing the AT-06 collision guard.

`ResumeScanner` walks the `sonnet-*/` directories, reads each `spec.json`, and marks a worker "complete" only if its `result.json` passes `SchemaValidator.Validate`. Missing or schema-invalid results trigger redispatch.

`ResumeCommand` replaces the stub in `Program.cs` and reuses the existing `SonnetDispatcher`/`ConcurrencyController` fan-out path, dispatching only the incomplete workers identified by the scanner.

One judgment call: `SonnetDispatcher` previously never called `PersistPromptAsync`; a call was added after `BuildRequest` so `prompt.txt` exists for every completed worker.
