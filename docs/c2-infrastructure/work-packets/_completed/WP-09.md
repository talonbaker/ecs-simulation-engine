## Summary (≤ 200 words)

Wired all prior pillars (Anthropic client, cache manager, batch scheduler, cost
ledger) into a single runnable `Warden.Orchestrator` binary with full mock-mode
support and a System.CommandLine-based CLI.

The key seam introduced is `IAnthropicClient`, an interface in `Warden.Anthropic`
that `AnthropicClient` now implements and `MockAnthropic` satisfies offline.
This keeps the mock fully in-process with no real API calls.

`ConcurrencyController` runs Sonnet workers via `Task.WhenAll` + `SemaphoreSlim(5)`
and preserves input ordering in the result array. `SonnetDispatcher` handles
prompt assembly, retry (Polly v8 exponential backoff, 3 retries, 429/5xx only),
ledger writes, and budget checks. `HaikuDispatcher` is a thin delegator to the
existing `BatchScheduler`.

`RunCommand.RunAsync` is the testable core: it resolves globs, validates specs
against JSON Schema, guards the API key, fans out Sonnets, collects
`ScenarioBatch` results, fans out Haiku via the batch API, and returns a
structured exit code (0/1/2/3/4).

The mock harness (`MockCannedResponses` + `MockAnthropic`) loads canned
`*.sonnet.json` and `*.haiku-NN.json` files from `examples/mocks/`, enabling
full end-to-end test coverage without network access.

## Design notes

**IAnthropicClient placement:** Interface lives in `Warden.Anthropic` (not
`Warden.Orchestrator`) so the real `AnthropicClient` can implement it without a
circular dependency. Tests and orchestrator both reference `Warden.Anthropic`.

**Mock specId extraction:** `MockAnthropic.CreateMessageAsync` has no structured
call context, so it extracts the `specId` from the serialised user-turn text via
regex `"specId"\s*:\s*"(spec-[a-z0-9-]+)"`. This is intentionally fragile — it
only needs to work for canned smoke tests, not production.

**parentBatchId override:** Canned haiku files carry a static `parentBatchId`.
`MockAnthropic.StreamBatchResultsAsync` overrides it with the runtime-generated
`batch-mock-NN` ID (using `with { ParentBatchId = batchId }`) so schema
validation in `BatchScheduler` passes.

**NullChainOfThoughtStore:** Declared here; WP-10 replaces it with a real
file-based implementation. The interface is in `Dispatcher/`, not `Mocks/`,
because it is part of the production contract.

**RetryPolicy zero-delay:** `RetryPolicy.Build(TimeSpan.Zero)` is used in tests
(and in mock/dry-run mode) to make retries instant and keep test duration under
1 s.
