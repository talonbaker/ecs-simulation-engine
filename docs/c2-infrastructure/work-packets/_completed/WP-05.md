## Beta headers added and rationale

Both headers are set in `Warden.Anthropic/Internal/HttpClientFactory.cs`:

```
anthropic-beta: prompt-caching-2024-07-31,message-batches-2024-09-24
```

| Header value | Feature enabled | Why required |
|:---|:---|:---|
| `prompt-caching-2024-07-31` | `cache_control` field on content blocks | Without this header, the API ignores `cache_control` markers and charges full input token rates for every call. Pillar D.1 requires caching; omitting the header silently defeats it. |
| `message-batches-2024-09-24` | `POST /v1/messages/batches` and `GET /v1/messages/batches/{id}` endpoints | Without this header the batch endpoints return 404. Pillar D.2 requires batching all Haiku calls; omitting the header makes the Batch API unavailable. |

**Note:** At the time of implementation (2026-04-24), both features were still behind beta headers per the Anthropic documentation. If Anthropic graduates either to GA and removes the requirement, delete the relevant value from `BetaHeaders` and update this note. The constant is isolated in `HttpClientFactory.cs` for exactly this reason.

## Key design decisions

**`ModelId` private constructor via inner JSON converter.** A nested `private sealed class ModelIdJsonConverter` can call the private `ModelId(string)` constructor (C# inner-class access rules), while external code sees no public parameterised constructor. The JSON converter handles both directions: write = emit the raw name string; read = call the private constructor with the wire value (allowing any name from the API, not just the three known ones).

**`ContentBlock` as abstract record with `[JsonPolymorphic]`.** Follows the exact pattern established in WP-02's `AiCommand`. Each derived record (`TextBlock`, `ToolUseBlock`, `ToolResultBlock`) lists all its properties including the optional `CacheControl` as primary constructor parameters with default `null`. This avoids the init-only setter problem for inherited base properties.

**Snake_case via `[JsonPropertyName]`, not a custom naming policy.** `JsonOptions.Wire` uses camelCase naming. The Anthropic API uses snake_case. Rather than override the options (which would break all other Warden serialisation), each multi-word property carries an explicit `[JsonPropertyName]` attribute. Single-word properties (`Model`, `Role`, `Id`, etc.) are identical under either policy.

**`JsonSnakeCaseEnumConverter<T>` in `Warden.Anthropic`.** A .NET 8-only type (uses `JsonNamingPolicy.SnakeCaseLower`). Kept in `Warden.Anthropic` rather than `Warden.Contracts` because `Warden.Contracts` targets `netstandard2.1` which does not have `SnakeCaseLower`. Used solely for `BatchProcessingStatus`.

**`StreamBatchResultsAsync` fetches results via `results_url` from `GetBatchAsync`.** The WP acceptance test (AT-04) specifies "via the `results_url`". The method calls `GetBatchAsync` once to resolve the URL, then streams the JSONL. The orchestrator's polling loop is the right place to call `GetBatchAsync` repeatedly; this method assumes the caller has already confirmed the batch is ended.

**One `HttpClient`, handler injection.** The only `HttpClient` field is `_http`. Tests inject a `StubHandler` through the constructor to avoid network calls. `disposeHandler: false` is passed when wrapping an injected handler so the client does not own the handler's lifetime.
