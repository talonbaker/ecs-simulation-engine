## Key design decisions

**Enum serialisation strategy.** Three policies are in play across the schemas:
- *Global camelCase* (`JsonStringEnumConverter(JsonNamingPolicy.CamelCase)` in `JsonOptions.Wire`) handles `OutcomeCode`, `SpeciesType`, `WorldObjectKind`, `DeliverableKind`.
- *Kebab-case* (`JsonKebabCaseEnumConverter<T>` attribute on the enum type) handles `BlockReason`, `VerificationKind`, `AssertionKind`, `ScenarioCommandKind`.
- *PascalCase verbatim* (`[JsonConverter(typeof(JsonStringEnumConverter))]` no policy) handles `DominantDrive` to match the schema's `"None"/"Eat"/…` values.

**Minimal validator keyword scope.** Supported: `type`, `required`, `properties`, `additionalProperties`, `items`, `enum`, `const`, `minimum`, `maximum`, `minLength`, `maxLength`, `minItems`, `maxItems`, `maxProperties`, `pattern`, `oneOf`, `allOf`, `if`/`then`, `$ref`, `$defs`. Unknown keywords throw `NotSupportedException` at schema-load time (fail-closed). External `$ref` (e.g., `sonnet-result → sonnet-to-haiku`) is intentionally treated as "accept any value" since the embedded validator has no cross-file resolution.

**`AiCommand` polymorphism.** Uses `[JsonPolymorphic(TypeDiscriminatorPropertyName = "kind")]` with six `[JsonDerivedType]` registrations. Discriminator is written first in serialised output; `PropertyNameCaseInsensitive = true` handles round-trip parsing.
