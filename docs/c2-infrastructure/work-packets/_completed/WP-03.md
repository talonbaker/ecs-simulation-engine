## Key design decisions

**`capturedAt` added as explicit parameter.** The WP-03 spec signature omits it, but the design notes state "capturedAt is passed through from the caller — it never calls `DateTime.UtcNow`." The parameter is required for determinism (AT-02). Added as a required `DateTimeOffset` before `tick`.

**`EntityManager` overload for species classification.** The canonical signature `Project(snap, capturedAt, tick, seed, simVersion)` falls back to `SpeciesType.Unknown` for all entities. The extended overload `Project(snap, em, capturedAt, tick, seed, simVersion)` builds a `Guid → Entity` lookup and uses `SpeciesClassifier` for tag-based resolution. AT-03 tests use the extended overload; callers without a live manager use the fallback.

**Minimal APIFramework touch.** `EntityTemplates.SpawnHuman`/`SpawnCat` each gained one line (`entity.Add(new HumanTag())` / `entity.Add(new CatTag())`). Both struct types pre-existed in `Tags.cs`. This is surfacing existing types, not adding new component kinds (consistent with non-goal 2).

**`DriveComponent` may be absent at spawn.** `BrainSystem` writes `DriveComponent` on the first tick; a freshly spawned entity has none. `ValidateForceDominant` does not check for `Has<DriveComponent>()`; `HandleForceDominant` uses `entity.Has<DriveComponent>() ? entity.Get<DriveComponent>() : default` to handle the pre-tick case safely.

**`set-config-value` path navigation via reflection.** Dotted paths like `"Systems.Brain.SleepMaxScore"` are resolved by walking `SimConfig`'s property graph with `BindingFlags.Public | BindingFlags.Instance`. The final property is set directly on the live config object — identical semantics to `SimulationBootstrapper.ApplyConfig`'s `MergeFlat`, which also mutates in-place.

**Atomic batch rejection.** Full validation pass runs before any mutation. If any command fails, the entire batch is rejected (`Applied = 0`, `Rejected = batch.Commands.Count`). No partial application ever occurs.
