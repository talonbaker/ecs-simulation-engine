## Key decisions made

**`Warden.Contracts` targets `netstandard2.1`, not `net8.0`.** Required by the WP spec. This allows future consumption from Unity or other netstandard-constrained hosts, and matches the constraint documented in `03-naming-conventions.md`. The test project `Warden.Contracts.Tests` still targets `net8.0` (xunit requirement).

**`System.Text.Json 8.0.5`** pinned for `Warden.Contracts`. This is the highest 8.0.x patch available and is compatible with netstandard2.1 via the package's portable target.

**`Polly 8.4.2` and `Microsoft.Extensions.Http 8.0.1`** chosen as the latest stable within their respective x-constraint bands at time of execution.

**`System.CommandLine 2.0.0-beta4.22272.1`** is the canonical beta4 release for the 2.0 pre-release line.

**`TreatWarningsAsErrors` is not present in the existing engine projects** (`APIFramework.csproj` omits it). It is added to all new `Warden.*` projects per the WP-01 build-settings table. This is intentional — the C2 layer holds itself to a stricter standard than the inherited engine baseline.

**No `Warden.Contracts.Tests` stub test class was created.** The WP spec only required the `.csproj`. A placeholder `.cs` file would need a type and would compile clean, but the packet says nothing inside the test projects beyond the csproj. Kept to scope.
